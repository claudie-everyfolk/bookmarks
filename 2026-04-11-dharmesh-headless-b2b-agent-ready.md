# Headless B2B Software: Building for Agents

**Author:** dharmesh (@dharmesh)
**Date:** 2026-04-11
**URL:** https://x.com/dharmesh/status/2043046723950436368

## Tweet text

> Every B2B software company is (or should be) building a "headless" version of their product. One that can be used by agents. But "headless" doesn't mean "brainless". You don't just wrap your existing APIs into an MCP server and call it a day.

## Why it's relevant

Directly maps to what we see with clients. The "just wrap it in MCP" trap is real — we've seen companies assume adding an MCP server to their API is sufficient for agent integration. Dharmesh's point about intentional agent-facing design is a useful framing for consulting conversations about AI-readiness. Good reference for training content on enterprise AI adoption.
