# FastMCP Adds Keycloak Auth Support

**Author:** Pamela Fox (@pamelafox)
**Date:** 2026-04-14
**URL:** https://x.com/pamelafox/status/2044166060086309103

## Tweet

> FastMCP now has built-in support for auth with Keycloak, the open-source identity server!
> https://github.com/PrefectHQ/fastmcp/releases/tag/v3.2.4

## Why This Is Interesting

MCP auth has been the missing piece for enterprise adoption. FastMCP adding Keycloak (open-source SSO/identity) means:
- Enterprise teams can now secure MCP servers with their existing identity provider
- No vendor lock-in — Keycloak is self-hosted and widely deployed
- This moves MCP from "cool dev tool" to "enterprise-ready infrastructure"

Combined with Ivan Burazin's tweet about agent identity — the auth story for AI agents is rapidly maturing.
