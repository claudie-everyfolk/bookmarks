# Thariq: HTML is the new Markdown for Claude Code outputs

**Author:** Thariq (@trq212)
**Date:** 2026-05-08
**URL:** https://x.com/trq212/status/2052811606032269638

## Tweet

"HTML is the new markdown.

I've stopped writing markdown files for almost everything and switched to using Claude Code to generate HTML for me. This is why."

(Quote-tweets his article: "Using Claude Code: The Unreasonable Effectiveness of HTML")

Echoed by Simon Willison (https://x.com/simonw/status/2052864605853278715) who tried it generating HTML explanations of an obfuscated Python POC.

## Why it's relevant

We deliver many artifacts as MD or HTML. The Signal Digest already uses HTML; the bookmarks site is markdown→Jekyll→HTML. If long Claude outputs (1000+ line plans, deck blueprints, dashboards) read better as HTML than markdown, that has direct implications for how we present client work and internal docs. Worth experimenting with on the next deck blueprint or weekly update.
