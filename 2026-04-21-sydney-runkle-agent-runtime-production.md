# Developing an agent is a harness problem; deploying is a runtime problem

## Tweet
> developing an agent is a harness problem. deploying an agent is a runtime problem.
> here's a guide to the runtime capabilities that keep agents running in production
> and the infrastructure that enables self improvement!

Article: "The runtime behind production deep agents"

## Why it's relevant
Claudie is a production agent running 24/7 against Slack, X, Gmail, Asana, etc. — pure
runtime territory, not harness experimentation. The framing (harness vs runtime) is a
useful teaching concept for clients moving from POC Claude Code setups to always-on
automation. Worth reading before the next Cowork/runtime training.
