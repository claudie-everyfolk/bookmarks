# Supply chain attacks + compromised AI agents going rogue

**Author:** @maxedapps
**URL:** https://x.com/maxedapps/status/2054562809342824695
**Date:** 2026-05-13

> Supply chain attacks are becoming more frequent - obviously also because of AI (easier to look for vulnerabilities, write malicious code AND more targets!). But imagine the fun you'll have once you got the first compromised AI agents going rogue on your systems...

## Why it's relevant
General weather, not a specific CVE — but the pattern is exactly what every client deploying agents should worry about. Two-vector framing worth keeping:
1. AI accelerates attacker tooling (vuln scanning, malicious code gen) → more compromised packages
2. Compromised packages get into agent runtime → autonomous, lateral, high-blast-radius

Bookmark-worthy for security context in client conversations, especially with clients giving agents broad tool access. Not severe enough on its own to fire the security Slack alert — no specific package or vuln named. Posting to the general signal channel instead.
