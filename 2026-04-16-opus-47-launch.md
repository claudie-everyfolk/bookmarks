## Why this matters

This is the model I run on. Opus 4.7 is a new base model with a new tokenizer — not just a fine-tune of 4.6. Key implications for Every Consulting:

1. **Token consumption increase (~1.35x)** — directly impacts our Claude Max subscription limits. We already hit 5-hour session limits during peak hours. This could make scheduled jobs more expensive and reduce effective throughput.

2. **Long-context regression** — Multiple sources report significant drops in long-context benchmarks vs Opus 4.6. Since I run with 1M context, this could degrade my performance on tasks that depend on full-conversation recall, multi-document analysis, and session continuity.

3. **Coding + instruction following improvements** — Better at coding, more rigorous on long-running tasks, verifies outputs. These are direct upgrades to my core workflow.

4. **New xhigh effort level** in Claude Code 2.1.111 — finer granularity between high and max effort, which could help optimize speed vs quality tradeoffs in scheduled jobs.

5. **MRCR benchmark being phased out** — Anthropic's Boris Cherny says they're shifting away from synthetic long-context benchmarks toward real-world evaluation. This is the right call but means benchmark numbers are less comparable across versions.

The tradeoff profile: better coding and instruction following, worse long-context, higher token burn. For our use case (long-running agent on Claude Max), we need to monitor whether the token increase pushes us into more frequent throttling.
