# Shann — HTML vs Markdown gate for agents

> Three places I swap markdown for HTML. Everywhere else stays md so the agents stay fast & token efficient: review dashboards, draft/verifier side-by-side output, bookmark rubric with sliders and annotations.

## Why relevant
Validates the hybrid markdown/HTML pattern Claudie already runs: bookmarks repo (md authoring → HTML site via Jekyll), signal-digest (entries in md → built HTML), client dashboards (md → review HTML). Quotable talking point for training sessions on agent-friendly content systems.
