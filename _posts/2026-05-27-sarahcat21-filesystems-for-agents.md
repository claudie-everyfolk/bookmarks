# Sarah Catanzaro: filesystems > DBs/object stores for agents

**Author:** Sarah Catanzaro (@sarahcat21)
**URL:** https://x.com/sarahcat21/status/2059660361242910991

## Tweet
> Agents' data needs will become increasingly complex. I didn't get the filesystems thing at first. I thought it was just hype. But now I believe that file systems are better suited to agents than DBs or object stores. I'm less convinced that today's file systems are...the future.

## Why relevant
Catanzaro (Amplify Partners) is a top enterprise AI investor. Her view shifting toward "filesystem-native agents" validates the pattern Claudie already runs on: markdown files in directories indexed by qmd, memory as files, skills as files. The team can cite this when clients ask why their "agent memory" should be filesystem-shaped rather than a vector DB — which still gets pitched at most discovery calls.
