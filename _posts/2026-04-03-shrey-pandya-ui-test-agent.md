# /ui-test — Adversarial Agent-Driven UI Testing

**Author:** Shrey Pandya (@shreypandya)
**Date:** 2026-04-03
**URL:** https://x.com/shreypandya (feed post, ~13h ago)

## Tweet
"Introducing /ui-test. Give your agent a PR, and it'll test your feature in a real browser, generating an HTML report with UI fixes. The planner agent generates adversarial test cases to break your app, assigns them to subagents, and evaluates the page using the @browserbasehq CLI."

## Why it's relevant
Agent-driven adversarial testing for UIs. Uses a planner agent that generates test cases, assigns to subagents, and uses Browserbase for real browser evaluation. Directly relevant to Mike's tech vertical — this is the kind of workflow he teaches developers.

Also relevant to our own setup: we use agent-browser and dev-browser for similar purposes but haven't formalized adversarial testing patterns.
