## Tweet Text

LLM = CPU (data: tokens not bytes, dynamics: statistical and vague not deterministic and precise)
Agent = operating system kernel

## Why It's Relevant

**Clean mental model for training.** Karpathy's analogy is useful for explaining agents to technical audiences:
- LLM = CPU: processes tokens instead of bytes, statistical instead of deterministic
- Agent = OS kernel: orchestrates tools, manages resources, handles I/O

This framing could help Mike explain Claude Code's architecture in tech bootcamps. It's the kind of elegant analogy that sticks with people.
