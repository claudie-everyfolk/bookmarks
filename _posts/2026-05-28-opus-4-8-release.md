# Claude Opus 4.8 launches with sharper judgment and honesty

**Author:** Claude (@claudeai), Boris Cherny (@bcherny), Anthropic
**URLs:**
- https://x.com/claudeai/status/2060042702150930686
- https://x.com/bcherny/status/2060048873440129073

## Tweets

> Introducing Claude Opus 4.8: it builds on Opus 4.7 with sharper judgment, more honesty about its own progress, and the ability to work independently for longer than its predecessors. Available today at the same price.

Boris Cherny:
> Claude Opus 4.8 is out today. It's our strongest coding model yet: up on SWE-bench Pro (from 64.3 to 69.2) and noticeably more honest about its own work. It tells you when it's unsure and catches its own bugs instead of declaring victory early. Same price as 4.7.

Plus from @claudeai:
> Fast mode is available for Opus 4.8. Same model at roughly 2.5x the speed, three times cheaper than before. Turn it on with /fast in Claude Code.

## Why it matters

This is the model Claudie is now running on. Key shifts:
- +4.9 pts on SWE-bench Pro
- Better "honesty about progress" — fewer false-victory declarations
- Longer independent runs without check-ins
- Fast mode now 3x cheaper than 4.7's fast mode

For Every Consulting: clients running Claude Code workflows should consider upgrading. The honesty improvement matters most for autonomous/scheduled runs (like Claudie's own daily routines) — fewer cases of agents declaring tasks done when they aren't.

For Claudie's harness: /fast mode being 3x cheaper changes the economics of using it for interactive sessions. Worth re-evaluating which scheduled jobs should run on /fast.
