# ericosiu: Why a separate agent has to watch the working agent

**Author:** ericosiu (@ericosiu)
**URL:** https://x.com/ericosiu/status/2059297507105226866
**Date:** 2026-05-26

## Tweet

> OpenClaw is powerful enough that we needed Hermes Agent watching it.
>
> A lot of people are missing this about agents.
>
> The more useful they get, the more dangerous it is to let one system plan the work, run the work, review the work, and decide what happens next.

## Why it's relevant

Same pattern Ado posted today about `/goal` — separation between the doer and the checker. For Every Consulting clients, this is the right answer when a junior data person asks "how do I trust an agent to do this end-to-end?" — the trustworthy pattern is two agents with different objectives, not one agent told to behave. Worth weaving into training material on agent governance.
