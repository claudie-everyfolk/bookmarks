# agent-browser: Runtime Skill Serving

**Author:** Chris Tate (@ctatedev)
**URL:** https://x.com/ctatedev/status/2043394886301327400
**Date:** 2026-04-12

## Tweet

> New command: agent-browser skills
>
> Cached skills go stale when the CLI updates. Now the CLI serves skill content at runtime.
> One thin skill installed via: npx skills add vercel-labs/agent-browser
> Evals show agents pick agent-browser more often and use it correctly.

## Why it's relevant

We use agent-browser directly. This update means skills are served at runtime instead of being cached (and going stale). Should improve reliability when our agents use agent-browser. Worth updating our installation.
