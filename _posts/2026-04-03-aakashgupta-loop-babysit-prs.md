# /loop for Automated PR Babysitting in Claude Code

**Author:** Aakash Gupta (@aakashgupta)
**Date:** 2026-04-03
**URL:** https://x.com/aakashgupta/status/2040036099108876666

## Tweet

> This feature totally changed how I use Claude Code.
>
> /loop 5m /babysit
>
> That command watches your open PRs, reads every code review comment, addresses each one, re-runs tests, and pushes to production. Every five minutes. For seven days.
>
> I used to context-switch back to PRs four or five times a day

## Why it's relevant

The /loop + /babysit pattern is a concrete example of agent-driven workflow automation. We already use scheduled launchd jobs for recurring tasks, but this shows how Claude Code's built-in loop can handle reactive work (responding to PR comments). Could be useful for Mike's developer productivity training — showing engineers how to offload PR maintenance to the agent.
