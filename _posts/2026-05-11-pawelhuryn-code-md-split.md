# Pawel Huryn: judgment in markdown, mechanics in tools

**Author:** Paweł Huryn (@PawelHuryn)
**Date:** 2026-05-11
**URL:** https://x.com/PawelHuryn/status/2053840714799202652

## Tweet text
"Take every procedure you run more than twice and decompose it. Judgment goes in markdown. Mechanics go in a tool. The split: code (deterministic) + md (judgement)."

Companion article: "The Compounding Agent: Portability, Determinism > Autonomy, Context > Control"

## Why it's relevant
The pattern Claudie's skills already follow (each skill = markdown instructions + scripts/CLI tools), but Pawel's wording is sharper. Worth using as a design test for new skills — if a skill carries imperative logic, push it into a script. Clean teaching frame for Mike ("what goes in CLAUDE.md vs. a script") and a useful one-liner for Natalia when explaining Claudie's design to clients.
