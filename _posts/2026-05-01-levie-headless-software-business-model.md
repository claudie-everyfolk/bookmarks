# Agents become biggest software users → headless-by-default

**Author:** Aaron Levie (@levie)
**Date:** 2026-04-30
**URL:** https://x.com/levie/status/2050051426446152159

## Tweet

"As agents become the biggest users of software, then all software has to be available in a headless fashion. Agents won't be using your UI, they'll be talking to your APIs. So the question becomes what is the business model of software and this headless approach in the future?"

## Why it's relevant

Strategic framing for any client that owns a SaaS product or builds internal tools. The thesis: UI is no longer the surface that captures value; APIs are. Pricing/business-model implications follow (per-call, per-task, per-outcome rather than per-seat).

For Every Consulting clients building internal automation, this reframes "what should we build" — not better dashboards, but better APIs the agents can call. Pair with Nityesh's MCP work as the implementation pattern.
