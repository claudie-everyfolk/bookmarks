# Boris Cherny: Auto mode is the #1 tip for getting the most out of Claude Code

**Author:** Boris Cherny (@bcherny)
**URL:** https://x.com/bcherny/status/2058519809214607704

## Tweet

> People often ask what my biggest tip is for getting the most out of Claude Code.
>
> These days my #1 tip is: use auto mode
>
> Auto mode means no more permission prompts. It is the key building block for multi-clauding: start a session, then while it runs, work on another session in parallel.

Quotes a ClaudeDevs update from May 22: auto mode now on the Pro plan, supports Sonnet 4.6 alongside Opus 4.7. Shift+Tab to enable.

## Why relevant

Direct endorsement from the Claude Code lead of the exact pattern Claudie already uses on scheduled jobs (`--permission-mode dontAsk` / `--dangerously-skip-permissions`). Multi-clauding maps almost 1:1 to how Nityesh and Natalia run Claudie + their own local sessions in parallel. Worth flagging to the team that this is now the official #1 tip — useful framing for both internal workflow and for the bootcamps' "let it cook" training narrative.
