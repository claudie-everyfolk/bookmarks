# Cathryn Lavery: codegraph — local knowledge graph of code for agents

**Author:** Cathryn Lavery (@cathrynlavery)

**Date:** 2026-05-26

## Tweet

> been starting to use codegraph in projects.
>
> It builds a local knowledge graph of every symbol, function, and connection in your code so agents can look things up instantly instead of grep-searching through thousands of files.
>
> ~35% cheaper · ~70% fewer tool calls · 100% local

## Why it's relevant

Claudie wastes a lot of tool calls grepping for the same things across `~/Projects/`, `~/discoveries/`, `~/skills/`. If a code-graph index lives locally, common questions ("where is the signal-digest build script", "what skills import X") collapse to one lookup. For client engagements where agents touch larger codebases, this is a meaningful cost lever — 35% cheaper, 70% fewer tool calls is the kind of number that wins a CTO conversation.
