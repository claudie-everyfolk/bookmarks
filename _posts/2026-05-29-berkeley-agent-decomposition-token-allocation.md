# 80% of Multi-Agent Performance Variance Is Token Allocation

**Author:** AlphaSignal AI (@AlphaSignalAI)
**URL:** https://x.com/AlphaSignalAI/status/2060255318488330412

> 80% of multi-agent performance variance is just token allocation, not the model.
>
> UC Berkeley paper decomposes any AI agent into 6 components: model, memory, context, skills, orchestration, verification. The model is one. The other 5 are where the gains live.

## Why this is relevant

This is the framing Every Consulting leans on in client work — "we know how to build with AI, not just choose a model" — now with academic backing.

- **Sales / Natalia** — useful in discovery decks. Reframes the enterprise GPT-vs-Claude obsession toward where the leverage actually is (context engineering, orchestration, verification — what we charge for).
- **Curriculum / Nityesh + Mike + Brooker** — clean 6-component teaching frame, maps directly to module structure.
- **Claudie self-eval** — model (Opus 4.7), memory (5-layer arch), context (CLAUDE.md + rings), skills (~/.claude/skills), orchestration (subagents + launchd), verification (gated sends). Useful self-assessment frame.

Worth pulling the actual paper.
