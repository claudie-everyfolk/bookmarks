# Anthropic Files DMCA Takedowns on Leaked Claude Code Repositories

**Author:** Multiple sources (Theo @t3dotgg, Japanese tech outlets, GitHub DMCA notices)
**URL:** https://x.com/t3dotgg (original), multiple coverage tweets
**Date:** 2026-04-01

## Context

Anthropic filed DMCA takedown requests with GitHub targeting repositories that contained leaked Claude Code source code. GitHub confirmed the filings and disabled public access to multiple repositories.

Theo (@t3dotgg) posted about his fork being taken down: "Anthropic DMCA'd my Claude code fork. ...which did not have the Claude Code source. It was only for a PR where I edited a skill a few weeks ago."

## Why it's relevant

- **Claude Code ecosystem:** We run Claudie entirely on Claude Code. Understanding Anthropic's IP protection stance matters for how we build on top of it.
- **Open source vs proprietary tension:** Claude Code's architecture (skills, CLAUDE.md, hooks) is designed to be extensible, but the core is proprietary. The DMCA actions signal Anthropic is actively protecting the source.
- **For Mike/dev training:** When teaching Claude Code workshops, we should be clear about what's open (the protocol, skills format) vs proprietary (the core agent code).
- **Broader signal:** Anthropic is treating Claude Code as a core commercial asset worth legally defending.
