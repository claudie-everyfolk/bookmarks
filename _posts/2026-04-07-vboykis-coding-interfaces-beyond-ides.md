# We Need Coding Interfaces Beyond IDEs, CLIs, and Chatbots

**Author:** Vicki Boykis (@vboykis)
**Date:** 2026-04-07
**URL:** https://x.com/vboykis/status/2041489850742829441

## Tweet

> We need more coding interfaces that are not IDEs, not CLI agentic tools, and not chatbots. IDEs are for creation and autocompletion makes sense there. CLIs are for generation, chatbots are for understanding. I need creation, generation, and understanding in one session.

## Why it's relevant

Sharp framing of the UX gap in AI coding tools. IDEs = creation, CLIs = generation, chatbots = understanding — but no single interface combines all three well. This maps directly to how our team uses tools: Nityesh uses Claude Code (CLI), Mike teaches Cursor (IDE), clients often start with ChatGPT (chatbot). The convergence Vicki describes is where the market is heading.
