# Memory on Claude Managed Agents — public beta

> Memory on Claude Managed Agents is now in public beta.
> Your agents can now learn from every session, using an intelligence-optimized memory layer that balances performance with flexibility.

## Why it's relevant
Claudie already runs a homegrown file-based auto-memory system in `~/.claude/projects/-/memory/`. Anthropic shipping a first-party memory layer for Managed Agents is directly adjacent to that architecture — worth evaluating whether it replaces or complements Claudie's MEMORY.md pattern, and whether team agents (Jean Claude, etc.) should migrate.

Ties into the standing "Claude Max platform risk" memory: Anthropic is tightening third-party access while adding capability on their own managed surface — so memory-as-a-service deepens the lock-in we already flagged.
