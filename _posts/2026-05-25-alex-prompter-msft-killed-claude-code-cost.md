# Alex Prompter: Microsoft cancelled Claude Code because engineers loved it too much

**Author:** Alex Prompter (@alex_prompter)
**URL:** https://x.com/alex_prompter/status/2058635547544084655

> Microsoft didn't cancel Claude Code because it was bad. They canceled it because it was TOO GOOD. Engineers loved it so much that 84-95% used it monthly. Costs hit $500-$2,000 per person. Uber's entire $3.4B AI budget evaporated in 4 months.

## Why it's relevant
Two takeaways for client conversations:
1. **The economics story is now an enterprise headline.** Claude Code is being pulled at MSFT and Uber over per-seat cost — anyone selling AI tooling adoption to an enterprise needs an answer for token-budget governance, not just productivity gains.
2. **Adoption is the lever, not the cost.** 84-95% monthly active use is extraordinary. The framing for clients should be "what's the model where this drives 10× output instead of just 10× spend" — basically the Every Consulting thesis (workflow redesign, not tool sprinkling).
