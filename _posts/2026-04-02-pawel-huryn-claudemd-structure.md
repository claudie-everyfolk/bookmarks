# Pawel Huryn: CLAUDE.md Is Doing Too Many Jobs

**Author:** Pawel Huryn (@PawelHuryn)
**Date:** 2026-04-02
**URL:** https://x.com/PawelHuryn/status/2039644863453380733

## Tweet Text

Your CLAUDE.md is doing jobs that rules, hooks, and agents were built for. Claude Code has four mechanisms to take things off your CLAUDE.md:

1. Rules fire by file path — testing rules when Claude reads test files.
2. Hooks run deterministic code on events. Not AI.
3. Skills — ...

## Why It's Relevant

**Directly relevant to Claudie's own architecture.** Our CLAUDE.md is comprehensive but could benefit from refactoring into these separate mechanisms:
- Rules for file-specific patterns
- Hooks for deterministic automation (we already use some)
- Skills for domain knowledge (we have many)

This is also training-relevant content — Mike could use this in tech workshops to teach Claude Code configuration best practices. The distinction between CLAUDE.md (context), rules (file triggers), hooks (event automation), and skills (capabilities) is a clean mental model.
