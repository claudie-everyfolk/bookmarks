# Anthropic killed Markdown

**Author:** Felipe Demartini (@namcios)
**URL:** https://x.com/namcios/status/2053257337746657561
**Date:** 2026-05-10

## Tweet

> Anthropic just killed Markdown. A Claude Code engineer published an article yesterday that could herald the start of a new era.
>
> Markdown was never the right format for communication between humans and AI. The author himself admits he's never read an AI-generated Markdown file with more than 100 lines all the way through.
>
> Markdown assumes you'll read from start to finish. HTML assumes you want to see what matters and tinker with your hands.
>
> In practice: 30 project tickets become a draggable kanban with export button. Rate limiting logic becomes an SVG flowchart. Code review becomes a colorized diff with dependency graphs. Animation parameters get sliders with live previews. Project specs become 6 side-by-side interactive mockups.
>
> HTML is 2-4x slower to generate. But with 1M token context, that cost has vanished. And the generated HTML isn't just for humans — the verification agent reads it too. The spec stopped being a document and became shared memory between agents.
>
> Markdown is a report. HTML is an interface. Reports are for reading. Interfaces are for continuing the work.

## Why it's relevant

Foundational shift in how AI agents output deliverables. Every Consulting produces tons of markdown — dashboards, process maps, weekly updates. If the right artifact for client work is interactive HTML rather than long markdown reports, this changes how Claudie should structure outputs across nearly every skill: client dashboards, process maps, signal digests, weekly updates. Also implies agent-to-agent communication should be richer than markdown.
