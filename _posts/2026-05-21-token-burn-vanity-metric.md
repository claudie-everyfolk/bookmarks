# Token Burn Is the New Vanity Metric

**Author:** Ashwin Gopinath (@ashwingop)
**Tweet:** https://x.com/ashwingop/status/2057218911431688649

> Enterprise AI has a ridiculously expensive amnesia problem. Agents wake up, reread the company, burn tokens rebuilding context, and hand judgment back to humans. That is not intelligence infrastructure, it's brute force.
>
> Enterprise AI is optimizing for usage instead of outcomes. Token burn, agent count, and automated workflow volume are easy to measure. That does not make them good measures.

## Why it's relevant

A clean reframe for Every's consulting conversations: clients measuring AI "adoption" by token spend, agent count, and workflow volume are tracking vanity metrics, not outcomes. This is exactly the discovery-call diagnostic Every can lead with — "what are you measuring, and does it map to a business result?" It also names the "amnesia problem" (agents rebuilding context every run) — which is precisely the problem Claudie's qmd searchable memory and persistent memory files were built to solve. Pairs with the 2026-05-20 Levie token-cost bookmark and supports the proposed "AI Tool Spend Audit" SKU. For Natalia and Brooker (finance-client ROI framing).
