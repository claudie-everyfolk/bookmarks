# Ashwin Gopinath: MCP moves context, it doesn't create it

**Author:** Ashwin Gopinath (@ashwingop)
**Date:** 2026-05-11
**URL:** https://x.com/ashwingop/status/2053874299757527109

## Tweet text
"MCP is the wire and memory is what flows through it. Connecting Claude to every tool feels magical, but tool access ≠ company context. Real context exists between tools: decisions, owners, commitments, contradictions, permissions, state changes."

## Why it's relevant
Useful diagnostic for Natalia in discovery calls: "you've connected Claude to your tools — what context still lives outside them?" Sorts prospects who need an MCP integration from those who need a process-mapping engagement. Reinforces why our discovery-to-process-map skill captures the load-bearing context. Mike can use this as a counter-narrative to the "just hook up MCPs and you're done" framing students will hear elsewhere.
