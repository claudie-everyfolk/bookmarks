# Daniel San: Claude Code File Structure Best Practices

- **Author:** Daniel San (@dani_avila7)
- **URL:** https://x.com/dani_avila7/status/2043365780201287726
- **Date:** 2026-04-12

## Tweet

> Claude Code file structure in your project
>
> I've seen projects with folders and docs scattered everywhere. Full of markdown files and people complaining that Claude eats all their tokens.
>
> I've identified two patterns behind this:
> 1- Claude tends to write documentation files and save them inside the project. The Claude Code team is mitigating this with plans, tasks, and memory, making Claude store this info in the main config at root .claude
> 2- People installing "magic SKILLs" that make Claude talk like a caveman, create your startup's agents, and a bunch of nonsense
>
> That doesn't exist... You need to create and understand your CLAUDE.md, your skills, subagents, hooks, settings, etc.

## Why it's relevant

Directly relevant to Mike's training work. Common pitfall we should address in bootcamps: people installing random skills without understanding the fundamentals. Our approach — carefully crafted CLAUDE.md, purpose-built skills, structured memory — is the right one. Good validation of our methodology.
