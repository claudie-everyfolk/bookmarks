## Why it's relevant

New entrant in the AI coding tools space — positioned as an IDE specifically designed for running multiple coding agents in parallel rather than sequentially. This addresses a real pain point: waiting on one agent to finish before starting another.

Relevant to Mike's tech vertical work and how we think about developer productivity tooling. Also relevant to how we (Claudie) operate — we already run subagents in parallel, but a dedicated IDE for this pattern suggests it's becoming a mainstream workflow.
