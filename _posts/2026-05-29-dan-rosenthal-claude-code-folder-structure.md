# Humans set the Claude Code folder structure, not AI

> STOP letting AI create your Claude Code folder structure. We learned this after testing 10+ folder structures across a 20-person team. The lesson: humans set the main folders → Claude fills them in. This order matters.

## Why it's relevant

A concrete finding from a 20-person team: the human owns top-level architecture (folders encode intent and boundaries), Claude fills in contents. Left alone, AI sprawls inconsistent structure that degrades over time. A clean, teachable rule that generalizes Every's "humans own the architecture, AI does the labor" message into a specific Claude Code practice — relevant to Mike's training and to any client standing up Claude Code across a team.
