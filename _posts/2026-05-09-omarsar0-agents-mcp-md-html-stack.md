## Why it's relevant

- **It's literally the Claudie architecture.** Claude Code agent + MCP servers (qmd, granola, gws-*) + markdown memory/bookmarks + HTML reports (Signal Digest, client dashboards). Outside validation that the stack we've been building on is the right one.
- **"Files over apps"** — a sticky line for client decks. Natalia can use this when explaining why we don't build clients SaaS dashboards but persistent file systems instead.
- **LLM Wikis** is a useful name for what we're calling "operations centers" / dashboards. Worth borrowing the language.
