# Start with Cowork, then Claude Code — the sequence is the insight

**Author:** Aakash Gupta (@aakashgupta)
**URL:** https://x.com/aakashgupta/status/2057116178032853081
**Date:** 2026-05-20

> Most PMs learning Claude are asking the wrong question. "Cowork or Claude Code?" The frame is wrong. They run on the same repo. The same CLAUDE.md. The same folder structure. Whatever you build in one loads automatically in the other.
>
> The sequence is the insight. Start with Cowork — the interface strips out every friction point that makes Claude Code hard to learn. Files visible without plugins. HTML renders on click. Works from mobile, web, or desktop. The focus stays on how agents work, how to aggregate knowledge, how to define your workflows.
>
> Once that foundation is built, you add the terminal. Claude Code stops being a new tool — it becomes the same system with a more powerful interface layered on top.
>
> The mistake most PMs make is skipping Cowork. They go straight to Claude Code because it sounds more serious, then spend two weeks wrestling with VS Code setup instead of learning how to work with agents. The tool wins. The skill doesn't develop.

## Why it's relevant

A ready-made onboarding curriculum for Every's non-technical and finance-vertical trainings. The key teaching point — Cowork and Claude Code share one repo, one CLAUDE.md, one folder structure — reframes the choice as a sequence, not a fork. Brooker can use "start with Cowork, build the repo, learn the workflows, then layer in the terminal" as the spine of a non-engineer training track; it removes the VS Code-setup cliff that stalls beginners. Mike can position Claude Code as the upgrade, not the entry point. Note the caveat in the companion bookmark: Cowork's sandbox ignores managed settings — fine for individual learning, a gap to flag for managed org rollouts.
