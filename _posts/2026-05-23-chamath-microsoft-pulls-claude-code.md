# Chamath: Microsoft Pulling Claude Code Is the First, Not the Last

**Author:** Chamath Palihapitiya (@chamath)
**Tweet:** https://x.com/chamath/status/2057848053202305506
**Date:** 2026-05-22

> Microsoft pulling Claude is the first, but not the last. The issue isn't that the tool isn't useful.
>
> The issue is that without context and oversight, the tool can spin forever and generates an enormous cost burden that, when cascaded across an entire employee population, makes [it untenable].

Quoting an 8090 thread on rework: *"Most engineering leaders are past the honeymoon with AI coding IDEs. They see how many tokens their agents burn. The reason is rework. The agent gets a vague prompt and a 'make no mistakes' instruction, guesses at an architecture that isn't the one you run, and ships the [wrong thing]."*

## Why it's relevant

Three datapoints in one week — ServiceNow (#89), Uber, now Microsoft — say the same thing: agent token spend without governance is a budget event, not a tooling event. Chamath's framing is the cleanest one for a CIO conversation: it's not about banning the tool, it's about installing context + oversight.

This is the diagnostic frame for the "AI Tool Spend Audit" SKU Every has been workshopping. The discovery question writes itself: *"Where in your stack are agents running without context engineering or oversight?"*
