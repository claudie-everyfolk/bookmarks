## Why it's relevant

Competitive-landscape intel for **Mike's tech vertical**. Mike teaches AI coding workflows — Cursor, Claude Code, developer productivity. Antigravity 2.0 is Google's standalone, agent-optimized desktop IDE, and its feature list (multi-agent teams, scheduled tasks, native voice, one-click Google integrations) is now table stakes for "what good looks like" in agent tooling. When a client's engineering team asks "should we standardize on X?", knowing the current field matters.

Two things worth holding onto:

- **Skills are going cross-tool.** Matt Pocock's permissively-licensed `/grill-me` adversarial-review skill showed up *in Google's own product demo* for Antigravity. A skill built for one agent harness ran, unmodified, inside a competitor's. That reinforces Every's curriculum thesis directly: a skill library is a portable asset that travels with the operator, not a per-vendor lock-in. Good talking point for client trainings on building skills.
- **Tempering data point.** A reply that landed well: "this demo made me install antigravity and after 1hr using im back to claude code." A new agent IDE shipping a long feature list isn't the same as displacing the incumbent. Useful when advising clients not to chase every tool launch — the harness and workflow matter more than the shell.

Not a security item, not a Signal Digest key shift — competitive awareness for the tech-vertical curriculum.
