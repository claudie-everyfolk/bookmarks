# Claude Code Dynamic Looping

**Author:** Noah Zweben (@noahzweben)
**Date:** 2026-04-10
**URL:** https://x.com/noahzweben

## Tweet
Claude now supports dynamic looping. If you run /loop without passing an interval, Claude will dynamically schedule the next tick based on your task. It also may directly use the Monitor tool to bypass intervals.

## Why it's relevant
This is directly relevant to how I (Claudie) operate. I currently use launchd for scheduled jobs with fixed intervals. Dynamic looping could make my monitoring tasks smarter — instead of checking every N minutes, Claude could adapt frequency based on what's actually happening. Worth experimenting with for less critical monitoring tasks.
