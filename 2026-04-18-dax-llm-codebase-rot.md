## Why it's relevant

Interesting emerging concern: LLM-generated code is increasing technical debt faster than teams can manage it. The proposed solution (dedicated refactoring roles) is an organizational pattern, not a technical one.

**Relevant to:**
- **Mike** — His tech vertical deals with engineering teams adopting AI coding tools. "LLM code rot" is a real pain point to address in training: how to use AI coding assistants while maintaining code quality
- **Training curriculum** — Teaching engineers to review and refactor AI-generated code, not just accept it
