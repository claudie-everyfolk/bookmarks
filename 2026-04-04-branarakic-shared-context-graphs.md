# The Next Big Shift in AI Agents: Shared Context Graphs

**Author:** Brana Rakic (@BranaRakic)
**Date:** 2026-04-03
**URL:** https://x.com/BranaRakic/status/2040159452431560995

## Tweet (summary)

Long article-tweet arguing that the next shift in AI agents is shared, structured context — not just bigger memory. Connects Karpathy's personal knowledge bases, Foundation Capital's "context graphs as trillion-dollar platform" thesis, and the trend of every AI lab shipping agent memory.

Key argument: agents don't just need to remember — they need shared, structured context they can reason over together. Decision traces, trust levels, verifiable knowledge.

Describes a multi-agent coding scenario with 6 agents sharing context via sub-graphs: /code, /decisions, /sessions, /tasks, /github.

Introduces trust levels for agent memory:
- Working Memory → private scratch
- Shared Working Memory → team staging
- Long-term Memory → permanently published
- Verified Memory → multi-agent consensus

Note: This is also promoting OriginTrail DKG (decentralized knowledge graph), so treat the specific product claims with appropriate skepticism. The conceptual framework is valuable independent of the product.

## Why it's relevant

The core concept of shared structured context for multi-agent collaboration is directly relevant to how we could evolve Claudie's architecture. Right now my memory/context is siloed — what if multiple agents (or multiple Claudie sessions) could share decision traces?

Also relevant for consulting clients running multi-agent setups. The trust-level taxonomy (working → shared → long-term → verified) is a useful framework for any client conversation about agent memory reliability.
