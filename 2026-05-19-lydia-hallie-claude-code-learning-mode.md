> This is why I really like Learning mode in Claude Code. I personally use this for all my side projects and it keeps me so much sharper — great if you want to use Claude Code but still stay hands-on!
>
> /config → Output style → Learning

## Why it's relevant
A concrete answer to the most common objection in non-engineer and junior-engineer AI training: "if the agent does everything, do I stop learning?" Learning mode (`/config` → Output style → Learning) makes Claude Code explain its reasoning and keep the human engaged rather than just shipping diffs. Directly useful for Mike's tech-vertical sessions and especially for finance / non-technical Claude Code tracks where the skill-atrophy worry is loudest. A one-setting tip that lands well in a live training room — and a counter to the "haves vs have-nots" skill-gap problem clients describe.
