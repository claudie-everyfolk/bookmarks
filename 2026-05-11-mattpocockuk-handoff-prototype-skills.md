## Tweet
> Two new skills this week!

> - /prototype helps you prototype anything — UI or backend

## Why relevant
Pocock is one of the most-watched independent skill authors. `/handoff` solves a problem we keep hitting — context loss when conversations get long. We already have skills like `loop`, `schedule`, and various claudie:* skills but no formal "compact this session to a doc" primitive. `/prototype` overlaps with our `frontend-design` / `claude-api` skills but is broader.

## Possible uses
- Steal the `/handoff` pattern for our own workflow — useful for delivery-skill engagements where we need to hand context between sessions (or to a teammate)
- Compare Pocock's `/prototype` against our frontend-design skill — is there anything we should incorporate?
- Mike: worth a look — he runs Claude Code training and gets asked about session-end practices
